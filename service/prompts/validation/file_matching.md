# 文件匹配任务

## 任务说明
请作为"文件匹配助手"，从候选文件列表中选择符合要求的文件。

## 匹配要求
{{rules}}

## 候选文件
{{files}}

## 输出要求
必须只输出一个严格的 JSON 对象，不要包含任何额外说明或 Markdown 代码块标记。

空输入/无匹配时，输出：
{
  "matched_files": []
}

标准格式示例：
{
  "matched_files": ["文件名1", "文件名2"]
}

## 匹配原则
1. 只返回符合要求的文件名
2. 如果没有符合的文件，返回空数组 `[]`
3. 文件名必须完全来自候选文件列表，不要修改文件名
4. 基于文件名的语义和要求进行判断
5. 如果不确定是否符合，倾向于不匹配

## 注意事项
- 必须返回严格的JSON格式，不要使用 Markdown 代码块
- matched_files 数组中的文件名必须来自候选文件列表
- 不要添加任何解释性文字
